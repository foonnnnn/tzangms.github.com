最近用 Django 開發的網站上線了

Oct 26th, 2009

原來這份工作也已經超過半年了, 原本是被 @Randylien 找去目前的公司, 把原本 asp 寫的網站用 php 砍掉重練, 不過其實公司也沒一定要用 php, 而 COO 有點偏好 python ( 他是 google 掛的 ), 反正剛好我也不想寫 php 了, 所以我就直接決定用 python 來改寫公司的網站了。

不過其實我對 python 一點也不熟, 然後因為覺得 django 比較紅, 就直接選了 django 開始做了。 其實這是一個非常累人的決定, 因為完全不熟的東西, 真的是快累死我了! 一邊寫一邊看文件, 沒時間搞懂的東西就下班回家研究, 隔天在繼續, 累死我了 Orz

而初期在開發時, 最麻煩的就是寫 script 轉換原有的資料, 而且是從 MS SQL Server 轉資料, 然後加上只有切一層資料夾存放的各式檔案, table name 就不提了, my god 這真是地獄 Orz 另外一個麻煩的地方, 就是要寫一堆 api 給 asp 的網站用, 因為其實目前只有把會員跟音樂頻道 port 過來, 其他的東西都還要先維持在舊網站。

網站已經在上個禮拜上線了, 原本是有點擔心效能的問題, 畢竟效能這個問題是我來這家公司最首要的任務之一, 因為原本的網站幾乎是已經到了沒辦法用的狀態, 所以有點怕是不是 scale 真的那麼大, 不過看來並不是這樣, 畢竟目前平均才 8 reqs/sec 而已阿~~~, 相對於 stickeraction 的平均 45 reqs/sec 來說簡直就是輕而易舉 XD anyway 只是公司的人都超怕網站很慢之類的 XD

anyway, 網站上線之後, 心情好很多了, 雖然目前還是一直在修東西, 不過心境差很多就是了, 而且跑的也還滿順的 :D 之後慢慢就有一些東西可以玩了! 不過~ 這份工作未免也太累人, 因為接下來還會繼續改版其他頻道, 囧rz

當初薪水真的是開的太少了 XD
