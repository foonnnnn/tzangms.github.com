StickerAction 恢復正常!

Sep 2nd, 2008

因為前幾天的資料庫大爆炸, 導致 StickerAction 的快取都不見了, 所以主機因為沒辦法接受那麼大量的 request, 所以就先把偵測的功能關掉, 這兩天就一直在重寫程式的架構, 終於在今天把 StickerAction 給恢復正常了。

由於 StickerAction 最吃重的地方是在貼出去的貼紙, 貼紙的程式跑的時間越長, request 累積越多, 效能則是直線往下掉。 一開始在寫 StickerAction 的時候沒有考慮到太多效能的問題, 一方面也是因為很懶, 就想把工作一次做完, 所以其實一開始的架構根本就沒辦法負荷太多人貼貼紙。 一開始就只有對貼紙圖片加上一些讓瀏覽器快取的處理, 減少 request。

其實先前被 Yahoo 999 檔的時候, 就思考過整個架構的問題, 不過後來還是因為很懶, 而且被檔這事情, 也不是改了架構就可以解決的問題, 所以當時就只加上了對外存取頻率的限制功能。

其實只要一開始怕麻煩, 或是設計方向做錯, 就很難整個改掉了, 特別是在程式越來越龐大時, 就越不可能改架構, 然後會想一些方法來補救 XD (說一說, 好像跟說謊是一樣的, 撒一個謊, 要用更多的謊來圓….. :x )

就在這一次資料庫大爆炸之後, 我才下定決心要改架構 (握拳), 其實是不改根本不能跑 XD 因為對 cache 的依賴太重, cache 一清掉, 主機就爬不起來了。

總之, StickerAction 終於好了~~ Orz
