用 Fabric 來 Deploy

Oct 2nd, 2009

以前在寫 php 的時候是用 ruby 的 capistrano 來 deploy 程式, 雖然後來 php 好像有 phing, 雖然沒去深入研究, 可是光看文件就覺得沒有 capistrano 方便。 現在跑到 python, 當然就用 fabric 來 deploy, 雖然不像 capistrano 有 transaction, 不過也很夠用了。 ( 以下用的是 fabric 1.0.0a 的版本, 目前透過 easy_install 的版本好像還滿舊的 )

基本上 fabric 跟 capistrano 差不多 ( 應該就是學 capistrano 的 XD ), 寫個 fabfile.py 如下:

from __future__ import with_statement
from fabric.api import run, sudo
from fabric.context_managers import cd

PROJECT_PATH = '/var/www/myproject'

def dev():
    env.hosts = ['tzangms@localhost']

def prod():
    env.hosts = ['tzangms@tzangms.com', 'tzangms@tzangms2.com']

def deploy():
    with cd(PROJECT_PATH):
        run('hg revert . --no-backup; hg pull -u ')
        run("sed -i -e 's/DEBUG = .*/DEBUG = False/' settings.py")
        run('python manage.py migrate')
        sudo('service apache2 reload')

fabfile.py 大概像上面這樣, 接著就可以用下面指令來 deploy 到 dev 的環境:

fab dev deploy

或者 deploy 到 production:

fab prod deploy

真是超方便的 :p 不過當然這個 fabfile 簡化了許多實際上在 deploy 的手續, 不過應該可以描劃出基本 deploy 的方式, 以往用 capistrano 可以搭配 transaction 寫 rollback, 還有加上 deploy 版本的處理, 可以直接 rollback 版本, 不過後來超懶, 直接用 mercurial revert, 然後 pull -u, anyway, 有興趣的人在玩看看吧 :p

Mac Snow Leopard 安裝 Tornado

Oct 1st, 2009

tornado 官方網頁上寫的, Mac 10.5/10.6 要跑 tornado 需要先裝 setuptools, pycurl 跟 simplejson, 指令如下:

sudo easy_install setuptools pycurl==7.16.2.1 simplejson

( 不過其實我不太懂… 不是有 easy_install 就是有裝 setuptools 了嗎? )

不過我在 snow leopard 安裝 pycurl 失敗了, 所以 google 了一下, 將 ARCHFLAGS 改為 x86_64 就行啦

sudo env ARCHFLAGS="-arch x86_64" easy_install setuptools pycurl simplejson

接著抓 tornado 回來安裝, 就可以開始跑啦 :D

Virtualenv + Cron

Oct 1st, 2009

通常要進到某個 virtualenv, 都得先 activate 來啓用環境, 不過碰到 cron 要跑 virtualenv 裡的程式怎麼辦? dependencies 都在 virtualenv 裡, 所以 site wide 的 python 讀不到。 幸好剛剛 google 一下, 找到 google group 的某個討論, 原來直接用 virtualenv 裡面的 python 來跑就好了。

像是這樣:

/MY_ENV/bin/python /MY_ENV/myproject/manage.py generate_week_ranking

本來我還一直試著先去 activate, 沒想到這麼簡單壓~

Mac Mini 入手

Aug 31st, 2009

由於原本用的 Macbook 也已經用了約莫 2 年半, 慢慢的東西都快爛掉了, 入手不到一年, 光碟機就有挑片的狀況, 然後最近則是 LAN 會整個無法連上必須重開機才能恢復正常, 再來就是效能整個覺得 down 下來了, 所以想說該買台新機器了, 畢竟這台 Macbook 也已經兩年半了。

其實想買新機器很久了, 一直到這幾天真的受不了 Macbook 所以晚上就跑去優仕敗了這台 Mac mini。 我買的是便宜的那款 $21900, 1GB RAM, 120GB HD, 不過我把記憶體升級到 4G, 硬碟也換成了 500GB, 另外加上了新的 Apple Keyboard, 不過這樣還是比較貴的那一款 $28900 便宜, 不過當然還是要分期來付 XD

由於有換硬碟, 不過要節省時間, 所以我就叫店員幫我換好硬碟就好, 不須要安裝作業系統, 我回家直接灌 Snow leopard 就行, 順便試試 Snow leopard 是否真的如傳聞中說的, 不須要透過升級 Leopard 的方式也可以直接安裝。 哈, 果真可以直接裝ㄝ, 只能說 Snow leopard 只買 $1090 未免也太便宜了 :D

在換記憶體跟硬碟的同時, 店員給了我張優惠卷, 可以到二樓喝咖啡等機器裝好, 裝好會打電話給我, 還真是不錯, 不過最後要提一下, 那位店員在把東西換好, 把 Mac mini 的外殼裝回去之後, 居然發現有兩根螺絲沒有鎖上, 結果又拆了ㄧ次 ~”~ anyway, 是碰到新手吧 XD

總之目前跑起來, 比起 Macbook 真是舒服多了 :D
