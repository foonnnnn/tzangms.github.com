事情總要個開頭!

Apr 16th, 2007

最近在想件事情, 其實主要是從這次 Podcast 開始之後, 我體認到一個事情, 事情總要個開頭, 正所謂萬事起頭難。 起了頭之後, 你將會發現, 這件事情原來不是這麼的難, 怎麼我先前一直都沒發現呢! 將這件事情一直延宕, 但是… 其實我一直都知道阿!

每當朋友問起我, 要怎麼開始寫歌呢? 我總是告訴他們, 就拿著吉他, 隨便彈些和絃, 然後照著你所聽到的感覺亂哼! 那麼, 你的第一首歌就出來了, 當然! 一開始很難, 而且這首歌你也許會覺得很難聽~ 或者是太簡單, 甚至覺得很白目, 但是, 不要緊, 當你把這首歌記錄下來之後, 你就進入了一個新的境界, 沒錯, 你只要繼續寫, 第二首、第三首…慢慢的, 你會覺得, 寫歌不再是那麼困難, 搞不好還變成一個多產的創作者。 沒錯, 我總是這麼告訴別人的, 萬事起頭難阿! 但只要起了頭, 那麼你先前的疑慮將會一掃而空, 駕輕就熟, 嗯~ 但是對於這件事卻被我一直延宕, 我想… 該是時候要幫這件事情起個頭了, 不管好不好, 不然… 我想我還是會一直把它擱置下去… 所以最近要來開始籌劃一下, 嘿 ~ 祕密 :p

總之, 來起個頭吧!!

最近在研究的東西: CodeIgniter, Subversion

Apr 15th, 2007

前一陣子在找尋哪套 PHP framework 比較好用, 或者說是比較適合我用, 後來就基本上選定了 Code Igniter 開始玩, 基本上 Code Igniter 很快就可以上手, 要用framework 的其中一個重點, 就是可以縮短開發時程, 所以對於一些 Documents 很難懂, 或是要學很久的 framework, 我就不考慮了, 因為花太多時間去學習某套 framework, 這就有點違背了開始使用 framework 的原因了, 至少對我來說是如此。 當然選用 code igniter 也還有其他原因, 每套 framework 都有其 promise, 在選擇 framework 時, 看看你是否喜歡它的 promise、它的目標訴求, 喜歡就用, 沒有一定誰好誰壞。
