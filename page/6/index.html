關於 AWS EC2 Micro Instance

Sep 26th, 2010

玩了一兩個禮拜, 雖然價格很便宜, 跟一般的 VPS 差不多, 也許跟 Linode 512MB 的價格差不多, 也許有人會想說可以把 Linode/Slicehost 丟掉換用 AWS Micro instance, 可是… 人家 Linode 可是 4 顆 CPU, micro instance 只有一顆, 換過來穩掛。

其實今天剛好寫個玩具, 一分鐘會做一次 http request 然後寫入到 simpledb, 就這樣而已 Load Average 就一直在 0.41。 而相較於 stickeraction 來看, 目前用的是 Linode 768, 現在這個時間晚上 11 點, 每秒有 60 幾個 http request 進來, 可是 Load Average 只有 0.03。

所以說 micro instance 還是照他當初 announce 所建議的用途就好。

  1. DNS servers, load balancers, proxies, and similar services that handle a relatively low volume of requests.
  2. Lightweight cron-driven tasks such as monitoring, health checks, or data updates.
  3. Hands-on training and other classroom use.

如果有缺開發機的話, 我到覺得在 ap-southeast-1 開台 micro instance 來用挺不錯的, ssh 反應速度很不賴。 另外就是因為在 AWS zone 裡, 玩 simpledb 跟 AWS 其他的服務其實是有點優勢的, 像我現在就是在寫跑 cronjob 寫 simpledb 的玩具, 就這幾點來看還是有好玩的地方啦。

sorl.thumbnail 可能停止開發

Sep 13th, 2010

今天因為在寫某個東西, 需要在圖片上傳之後, 再視情況自動產生不同大小的縮圖, 而不能再上傳的時候就產生縮圖, 當時直覺想到的就是 sorl.thumbnail, 沒想到一進 sorl.thumbnail 的 google code 首頁就看到這個連結: Future。

其實 sorl.thumbnail 超方便的, 可是因為效能跟 scale 的問題, 所以當初再 SV 的網站上沒有大量採用, 就自己刻了一個縮圖的 library, sorl.thumbnail 只用在一小部分資料量非常少的地方。 先前 sorl.thumbnail 還是 django wiki 上列為最推薦的縮圖 library, 沒想到現在 …

稍微介紹一下好了, sorl.thumbnail 在 template 裡的使用方式大致上如下, 縮圖不存在的話, 則會自動依照給的尺寸、參數來自動產生縮圖

{% thumbnail photo.image 640x480 crop upscale %}

是真的很方便, 只是先前在 glusterfs 上面用, 暴慢, 雖然說應該歸咎於 glusterfs, 但是有些事情我無力去改變, 只好在多追一點效能, 捨棄 sorl.thumbnail 自己刻。

anyway, 另外值得一提的是, 在 python 處理圖像主要都是用 PIL, 而不像 PHP, Ruby 都用 Imagemagick 系列 ( 不要跟我提 GD 嘿 ), 可是在處理過幾百萬張圖片之後, 就會發現 PIL 其實有點廢, 不時就會來個 IOError, 後來找到其中一個原因是, 一堆有問題的照片都是 Adobe Photoshop 7 處理過的 ( EXIF 裡面有標 )

而 sorl.thumbnail 在 Future 一文當中提到下面這兩個 project, 甚至都比 sorl.thumbnail 好用

easy-thumbnails aino-convert

其中 aino-convert 則是用 imagemagick, 這個應該有趣多了, 最近得找時間來試一下。

常在 Dr. House 裡面聽到的 MRI

Sep 5th, 2010

MRI photo of Andrew Acomb

MRI 照出來大概看起來像上面這樣。 不過我拿到的檔案透過軟體來瀏覽是可以動的。

其實是上禮拜我爸到台北長庚醫院照 MRI 後, 拿回了一片光碟, 然後要我這禮拜帶著我的筆記型電腦回宜蘭, 要去給認識的醫生看 MRI 照出來的圖像, 看看身體狀況如何。

誰知道回來後一把光碟放進去 Mac mini, 卻看到的是 autorun.exe 而裡面有一堆 windows 的 dll, cab 等檔案。 anyway, 想說翻找一下資料應該可以直接找到我要的照片檔案, 直接打開來看好了, 沒想到怎麼都打不開。

最後乾脆開 terminal 用 vim 開啟這個檔案, 幸好在檔案裡頭有看到關鍵字「GE Medical Systems/DICOM Part 10 file」 後來就 google 了起來, 翻找了一陣子, 找到了 Mac 可用的 open source 軟體叫 OsiriX, 才把檔案打開。 我倒是從沒想到過也有跟醫療相關的 open source 軟體 :p

是說, 其實長庚醫院給的光碟裡面的圖片才 256x256 以及 512x512, 有沒有那麼弱阿?

最後, 醫生說 MRI 看起來沒有什麼太大的問題, 不過還是希望我爸身體可以健康一點就是了。
