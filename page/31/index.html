教召體驗

May 4th, 2008

這次教召一共4天, 簡短來說…就是超無聊 XD 不過也幸好不是在夏天去, 否則應該會更難渡過這漫長的教召生活。

其實教召讓我有種去當少爺的感覺 XD 這裡說的少爺, 其實是相較於新訓的時候, 爽的非常之多, 不過還是要集合、點名、洗餐盤…等等。

想當初在新訓的時候, 真的是非常非常的不自由, 連走路都覺得有壓力。 然而現在重回同一個營區, 但是感覺卻差非常多。集合..可以慢慢來… 等到大家都排好隊形了之後, 在慢慢走過去也不是問題。 上課要戴上鋼盔, 戴個一分鐘就拿下來, 也是ok的。所有當初新訓時的痛苦, 現在全都被大家當成了玩笑在開。 例如像是有人對連長講話沒加報告, 大家就會開玩笑的嗆他: 跟連長講話是不會加報告嗎?” 諸如此類的事。

生活作息還是跟當兵一樣, 5點半起床, 10 點就寢, 但是時間卻充裕, 因為不管是蚊帳或者是棉被都隨便折一折, 因為….之後會有阿兵哥幫你折 XD 而教召回去就是上課, 上課就是一天當中最難度過的時間, 因為大部分的時間其實都是在發呆, 而身邊的人也不認識, 所以上課時, 就是坐著很難坐的板凳乾等, 真的是很難熬。 不過慢慢的跟身邊的人熟了起來, 認識好多各種行業的人, 有人自己創業當老闆, 有人養魚、賣房子、做音響, 甚至還有人是當 Model。

說 Twitthat 抄襲的各位, 閉嘴吧!

Apr 23rd, 2008

首先~ 不想看長篇大論的人, 直接拉到文章中間, 有英文的地方開始看吧!

沒想到昨天寫一篇還是不夠讓那些說抄襲的人閉嘴, 反而在論壇跟書籤網站上歡樂了起來, 這些人目的為何? 老實說, 我完全不知道, 我只覺得我被罵得莫名其妙, 希望今天那些說 twitthat 抄襲 twitter 的人看到這一篇, 可以就此閉嘴。當然, 也僅限有理智的人, 對網路小白應該是無效的 XD

昨天文章的後半段, 列出了好幾個網站的截圖, 都是網站的 logo 都是 copy twitter logo 來改的 twitter 相關服務。 只是看來我昨天貼那些圖, 還是沒讓罵 twitthat 抄襲的人搞懂, 為何我要列這些網站給大家看, 又為何這些網站會直接就抓 twitter logo 的圖來改, 甚至是用相同的背景, 為何你們都不想想或求證?

一個服務的周邊, 當然希望跟這個服務風格一致, 這樣使用者才能無痛上手, 或是很快的熟悉這個衍生出來的服務, 當然這種情況, 也不只在 twitter 上發生, 在其他服務也是如此。

也許有高手會來看, 不過我就先假設大家沒開發過 twitter 的相關服務, 跟大家說明一下。

twitter 的使用者, 應該都有使用過 twitterfox, 或是最近很紅的 twhirl, 當用這些 client 發訊到 twitter 上, 在後面都會顯示 from xxxxx, 看下圖應該會比較清楚 (以下圖為例, 就是 from TwitterFox )

有開發過 twitter 相關服務的人就會知道, 要設定這個 from, 是需要跟 twitter 的開發人員聯繫, 寄 email 過去, 需要給他們說明, 網站顯示的名稱, 跟網址。

PHP 語法錯誤檢查

Apr 3rd, 2008

之前在 Tsung 那邊看到 大量檢查 PHP 程式是否有 Syntax Error 這一篇文章, 後來每次想到要檢查的時候, 都因為這一段太長記不下來 XD 所以我每次就又都從 Google 找這篇文章, 今天想說把它寫下來, 寫成一個 shell script 好了 XD

我把檔名叫 phplint #!/bin/bash find . -name *.php ! -exec php -l {} ;

在 chmod +x phplint 之後, 在想要檢查的目錄打 phplint 就可以了。

基本上我是在自己的 home 有開一個 bin 的目錄, 然後再 .bashrc 裡加上 export PATH=$PATH":~/bin"

之前的 twit shell script 也是這麼作 :p
