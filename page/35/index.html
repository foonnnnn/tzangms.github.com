以下感謝各位「推友」的祝賀, 再次祝大家新年快樂阿!! :D

新年快到了! 換個新版面吧!

Jan 28th, 2008

因為快過新年了! 所以也該換個版面吧? 總之最近應該會滿常看到這個 blog 一直變來變去的就是了。

會想換版面的主要原因, 其實因為看到了 Idea Grapes » 2008年的blog設計趨勢 這篇文章, 其中剛好提到了 the morning after 這個 wordpress 的佈景主題, 一看就滿喜歡的, 就興起一股想換佈景主題的念頭, 雖然還是很捨不得換掉先前的版面樣式。

其實先前很喜歡一個叫做 Revolution 的佈景主題, 他主要是設計給雜誌或新聞用的樣式, 不過 Revolution 這個佈景主題要付費購買, 最便宜的方案還要 $79.95 美元, 當然是不可能買來用阿!

不過前幾天看到這個 the morning after 這個佈景主題, 就覺得有點類似, 滿符合心中想要的樣式, 可以比較快速的改成自己想要的樣子, 雖然其中運用了許多 meta key 的運用, 才能跟 the morning after 所展示的樣子類似, 這點倒是麻煩了點 :p

在 Idea Grapes » 2008年的blog設計趨勢 這篇文章中也提到了 sifr 這個頗妙的東西。在設計網頁的時候, 我們只能用大部分電腦系統上通用、都有的字形, 以確保大家看到的設計樣式是一致的。 sifr 可以讓我們擺脫這個限制, 可以用你想要的字形來顯示你想要的字, 但使用者的電腦上卻不需要有這個字形, 其實就是用 javascript 跟 flash 來作這個動作, 有興趣的可以去看一看。

不過記得好多年前 netscape 也有某個解決使用者字形的方案, 就是在網頁上嵌入某段語法, 如果使用者沒有網頁所指定的字形的話, 將會自動下載。但是這兩種方法在中文上都可以算是無解。 排除標點符號…等不說, 中文不像英文只有 26 個英文字母阿! 所以不是一兩百k可以解決的事情, 而是好幾 MB 的事阿!!

最後的結論是, 我要來換版面了! 反正我的部落格也沒啥人在看, 所以可以隨心所欲一點 XD

Lighttpd Rewrite Rules for Wordpress Permalink

Jan 20th, 2008

先前由於把 Apache 換成 Lighttpd, 忘記還有 permalink 這東西, 所以 Wordpress 的 permalink 就掛點了。lighttpd 的 rewrite 好像沒有可以判斷檔案不在才做 rewrite 的條件, 又由於對 lighttpd 不熟, 只好在網路上找, 不過找到有些 rewrite rules 都嘛有問題, 所以參考一些找到的 rewrite rules, 然後修改成下面這樣 :p

url.rewrite = (
    "^/blog/?$" => "/blog/index.php",
    "^/blog/(?.*)$" => "/blog/index.php$1",
    "^/blog/(wp-.+)$" => "$0",
    "^/blog/([^.]+)/?$" => "/blog/index.php?$1",
)

目前跑的都還正常, 有問題麻煩請通知 XD

update: 用了好多天才知道, 這個 rewrite 有問題, 例如像下面這個網址重複兩個 blog 就掛點了

http://tzangms.com/blog/975

後來才重新看一下最後一個 rule, 把原本的 index.php/$1 改成 index.php?$1 就好了… hmm 我直覺上是這樣啦 XD anyway 總之這樣就正常了 XD
