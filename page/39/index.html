[Twitter] Twit From Command Line

Mar 10th, 2008

這是個寫來給像我這種 Programmer 用的(無用) twitter client (update only XD) 沒錯, 用 command line 也可以 twit 阿! XD 這個純粹是無聊之作, 不過這個應該滿適合像我這種整天都在 terminal 工作的 programmer。這個方法適用於 Mac 跟 Linux, 需求是系統有裝 curl 即可。

很簡單的, 首先把下面這段存成一個文字檔(shell script), 我是存成 twit, 然後修改 2, 3 行填上你在 twitter 的 username 跟 password 即可。

[source:php]

!/bin/bash

username=’your username’ password=’your password’ update_url=’http://twitter.com/statuses/update.xml’ message=$@

curl -s -u $username:$password -d status=”$message” $update_url > /dev/null [/source]

接下來執行 chmod +x twit 然後放到 /usr/local/bin 或是程式可以執行的 PATH。如果是在多人的環境下, 當然還是放自己 home 吧! XD

最後, 執行下面這一段就可以 twit 啦!

twit hello this is a twit message from command line

有看到你 twit 的訊息出現在 twitter 了嗎? 基本上只要 username 跟 password 沒填錯就可以用啦!

還有! 要發中文訊息的話, 記得將 locale 設成 UTF-8 勒! 還有~ 其實我不知道我會用多久 XD

[Mac] Dockdrop

Mar 8th, 2008

Dockdrop 真是個好東西阿! 只要把檔案拉到 Dockdrop 的 icon 上, 就可以直接上傳到你設定好的地方, 除了支援 FTP、SCP、WebDAV 之外, 還有 Flickr 勒!

而且在上傳完檔案之後, 還會直接把這個檔案的網址複製到剪貼簿, 滿適合快速分享檔案的。 像我的 Server 沒有開 FTP Server, 只有 SSH。 如果用 Dockdrop 就可以很方便的透過 SCP 把本地端的檔案傳到 Server 端。 不然還要開 FTP 軟體、等連線, 上傳完之後在關掉 FTP 軟體, 實在太繁雜。

如果設定多個上傳位址的話, 當把檔案拉到 Dockdrop 之後, 就跟上圖一樣, 會叫你選要上傳的地方(方式), 如果只設一個, 連問都不必, 而 Dockdrop 還是免費的! 真是個好物 XD

CodeIgniter Modular Extension: HMVC

Mar 8th, 2008

在使用 CodeIgniter 在開發的時候, 其實一直覺得 CI 的架構好像有點不足, 少了什麼東西。 例如說我想要有個使用者聯播的功能區塊, 有很多地方都想掛上這個功能區塊, 但是我該怎麼做? 在 CodeIgniter 現有的架構中, 看來就是寫成 Library 然後掛個 View, 然後把輸出 return 到呼叫的 Controller, 但是我總覺得這樣很亂, 而且東西一多, 會很難維護, Library 會不像 Library 而 View 到底是給 Controller 用還是 Library 用, 甚至 Model 也會一起捲進來, 這樣程式的架構會很雜亂。

所以後來我就自己動手 extends CI 開始寫起了 module 功能, 寫沒多久就發現! 原來早就有類似的東西, 而且在 CodeIgniter wiki 就有提到了, 就是 Modular Extensions - HMVC。

HMVC 可以用來幫 CI 加上 module 的功能, 我一直想要的就是這個功能阿!! 可以擺脫先前雜亂的方式, 可以很方便寫個模組化的功能。

簡單說明一下 HMVC, 在掛上 HMVC 之後就可以呼叫 module, 而 module 有自己的 Controller、View 跟 Model, 這樣一來寫好 module 之後, 直接在 Controller 呼叫 module 就可以了, 這個整個程式架構就非常的明朗阿!! 整個 module 的 code 都在同一個資料夾裡面, 非常清爽! 所以說 Framework 早該有 module 這功能的 XD

Lighttpd Rewrite Rules for CodeIgniter

Mar 1st, 2008

由於 CodeIgniter 使用 index.php 當作 font controller, 所以基本上把所有的東西都 rewrite 到 index.php 就可以了! 不過由於還是有 images, js 跟 css 檔案不能做 rewrite, 所以再加上類似於 public 那一行就行了! (我習慣是把 images, js 跟 css 資料夾都放在 public 資料夾裡面)

$HTTP["host"] =~ "tzangms.com" {
    url.rewrite = (
        "^/public/.*$" => "$0",
        "^/(.*)$" => "index.php/$1"
    )
}
