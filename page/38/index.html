Apache Rewrite Rules for CodeIgniter

Sep 29th, 2007

CodeIgniter 本來的網址看起來像是這樣

http://tzangms.com/index.php/profile/edit

不過可以用 mod_rewrite 改寫成這樣

http://tzangms.com/profile/edit

一開始在用 CodeIgniter 的時候為了要把網址弄漂亮一點, 在 mod_rewrite 上搞了很久, 雖然 CodeIgniter 的 document 寫的很棒! 但是 CodeIgniter URLs 這一篇裡面寫到關於 mod_rewrite 的部份就只有一點點。

為了要將 tzangms.com CI 化, 就免不了要再處理一次 mod_rewrite, 只是這次是在 godaddy 租的主機, 所以只能寫 .htaccess 了, 在處理上一篇的 No input file specified. 的問題 時, 看到了 CodeIgniter wiki 上的 mod_rewrite 章節, 這篇寫的很完整阿!!!

所以直接! 第一個步驟, htaccess 內容如下, done!

<IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteBase /
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteRule ^(.*)$ index.php/$1 [L]
</IfModule>

<IfModule !mod_rewrite.c>
    ErrorDocument 404 /index.php
</IfModule>

第二個步驟, 接著修改

system/application/config/config.php

找到下面這一行

$config['index_page'] = "index.php";

改成

$config['index_page'] = "";

沒錯, 這樣就樣就解決了阿!

[CodeIgniter] No Input File Specified.

Sep 28th, 2007

今天在將 tzangms.com CI 化的時候碰到了個問題, 就是一直出現下面這一行訊息

No input file specified.

第一次碰到這個, 完全沒頭緒! 原本以為是因為 mod_rewrite 沒設定好的關係, 所以才出現這個訊息。之後我找了好多 mod_rewrite 相關的資料, 還是沒辦法解決問題, 一邊修改設定, 一邊測試下面這個網址, 仍然還是一樣顯示 No input file specified.

http://tzangms.com/main

後來才發現我取消 mod_rewrite, 直接打下面這段網址也是出現相同的錯誤訊息。

http://tzangms.com/index.php/main

這時候才知道, 原來 mod_rewrite 沒設錯阿!! mod_rewrite 一直正確的運作著, 所以上面這兩個網址是一直指到同一個地方, 所以我才一直看到相同的訊息, 這才恍然大悟! main 一直有正確的 rewrite 成 index.php/main 阿!

Mac OS X 瘦身 : Monolingual

Sep 19th, 2007

最近我的 Macbook 硬碟真的是快用光光了…只剩下不到 5G … 所以我又抓了 Monolingual 來用, 可以拿來移除掉用不到的語系檔。

其實之前在 iBook 也用過 Monolingual, 不過現在的版本又多了可以移除不同平台架構的 Binary, 像現在我 Mac 上都只用到 Intel Binary, 而 PowerPC Binary 都嘛用不到, 所以…把 PowerPC Binary 都移掉吧!!! XD

語系檔的部份, 我只留了 Chinese (Traditional Han)、English 跟 English (United States), 其實應該 United States 應該也不用留, 可是我怕怕的 XD 因為…不管如何…絕對不能移除 English 阿!!! ok.. 在跑好一陣子之後, 清出了 2.7G 的空間!! :D

至於 Binary 的部份, 我只留 Intel, 其實…我是不是應該保留 Intel 64-bit ? 因為我的 Macbook 是 Core 2 Duo 阿! anyway, 在移除 Binary 的時候, 如果有用像是 MS Office 這種東西的話…還是不要亂移 XD 因為好像有些大程式還是會用到 Rosetta 這種東西… 反正怕就不要亂移就是了… 電腦掛掉也不要怪我 XD 怕的人可以看看這篇 Anyway 我只留了 Intel Binary, 又清出了 2.4 G 啦!!

哈哈哈! 空間整個多出 5G, 其實我最近覺得我的 Mac OS X 開機變很慢…所以專挑一些無聊事作 …

關掉 Mac OS X 的 Spotlight

Sep 18th, 2007

spotlight! 完全沒在用 XD 根據這個網頁的說明, spotlight 會在你複製、搬移檔案的時候做 index 的動作, 所以說, 沒在用 spotlight 的話, 不就浪費那些效能在做 index, 所以… 來移除 spotlight 吧! 應該說關掉 spotlight :p

1. 首先取得 root 權限啦!

sudo su -

2. 接著是修改 /etc/hostconfig

vi /etc/hostconfig

找到 SPOTLIGHT 這行, 把 -YES- 改成 -NO-, 如下

SPOTLIGHT=-NO-

3. 關掉 indexing

mdutil -i off /

會顯示

Indexing disabled for volume.

4. 移除 indexing

mdutil -E /

會顯示

Volume index removed.

5. 再來打開活動監視器結束掉 mdimport 這個程序

6. 最後是回到 / 移除掉 Spotlight 檔案

cd /

rm -rf .Spotlight*

7. 好! 重開機!!

不過後來我發現… 連用 Finder 右上角的搜尋框也找不到東西了 XD 所以說~ 這個東西, 無聊的人可以試一試 XD 還有電腦搞壞不要怪我 XD
